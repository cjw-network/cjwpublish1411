    Features:
    1. Moves all the controls for the current page to the top to avoid scrolling.
    2. Hides right menu by default to hide features that are not relevant for a editor (or even most admins) during normal use.
    3. Slims down the top header considerably to get more space for content area, also implements support for small screens (down to 200px) by automatically wrapping the header.
    4. Use tabs for misc object details (Preview,Details,...) to save scrolling to sub items and screen space.

    Other ideas:
    1. Add context menu to sub items on far right side when you hover over the line (like dropbox web interface)
    2. The final result will need to be able to work correctly (with page reloads) for tabs, size control and show/hide controls if javascript is disabled or not supported.
    3. Always refer to Nodes and Objects as "Pages" and if placement specific: "Placement". And (Content Object) Class should refer to as "Page type" or "Type" for short. This way only developers will need to learn what a Node, Object and Class is!
